# frpdeck 1.1.1

`1.1.1` is a small patch release focused on finishing the 1.1.x cleanup work. It keeps the existing feature surface intact while making MCP startup behavior, machine-readable CLI output, and internal command wiring more predictable.

## Highlights

- Stabilized the default Python interpreter embedded into generated MCP stdio wrapper scripts.
- Kept CLI `--json` output isolated from instance console logging so automation receives clean JSON.
- Further reduced internal duplication in MCP tool registration and CLI orchestration paths.

## Notable Changes

- `frpdeck mcp install-stdio-wrapper` now defaults to embedding the Python interpreter that is running `frpdeck` when the wrapper is generated.
- `--python` remains the explicit way to embed a different interpreter into the wrapper.
- MCP tool registration keeps the same public tool set and schemas, but bound and generic wrapper construction now share one shape-driven helper path.
- The 1.1.x cleanup around `proxy`, `apply`, logging, and HTTP request headers is included in this patch release.

## Upgrade Note

If you regenerate MCP stdio wrappers on hosts that previously relied on `VIRTUAL_ENV` implicitly changing the embedded interpreter, that behavior is no longer automatic.

Use `--python /path/to/python` when you need a specific interpreter that differs from the one currently running `frpdeck`.
