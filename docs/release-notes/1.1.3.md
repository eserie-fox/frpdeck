# frpdeck 1.1.3

`1.1.3` is a patch release focused on permission handling, release metadata consistency, import-boundary cleanup, and logger/MCP stability. It rolls up the post-`1.1.2` work on write-command privilege handling, dynamic version sourcing, package export cleanup, and the bound-instance logging fix.

## Highlights

- Mutating commands now share a consistent `--sudo` full-command re-exec path and clearer fail-fast permission errors.
- Package version metadata now comes from `frpdeck.version.__version__`, and package `__init__.py` files no longer act as compatibility export layers.
- Bound MCP stdio startup and other instance-scoped logging paths now keep `frpdeck.log` pinned to its stable symlink name instead of growing daily log filenames across restarts.

## Notable Changes

- Write commands now support `--sudo` as a true full-command re-exec flag, so non-root users can retry mutating workflows through sudo before instance loading or filesystem mutation starts.
- Permission-related failures now surface more consistently across initialization, rendering, syncing, restarting, upgrading, uninstalling, proxy mutation, and MCP wrapper install or uninstall flows.
- The package version is now sourced dynamically from `frpdeck.version.__version__`, which keeps release metadata and package builds aligned to one canonical version definition.
- Package-level `__init__.py` modules are now namespace markers only. They no longer re-export leaf-module symbols or provide lazy compatibility aliases.
- Instance logging now keeps `frpdeck.log` as the stable entry path and does not resolve that final filename to the current daily log target during configuration loading.
- Daily log rotation continues to write `frpdeck-YYYY-MM-DD.log`, but it now receives the stable symlink name that it expects as input, which restores stable bound MCP stdio startup for instances with existing log symlinks.

## Upgrade Note

- If external Python code imported symbols from package-level entry points such as `frpdeck`, `frpdeck.config`, `frpdeck.logging`, or `frpdeck.mcp`, switch those imports to the leaf modules directly. Examples: use `frpdeck.version`, `frpdeck.config.instance`, `frpdeck.logging.daily_symlink`, and `frpdeck.mcp.server`.

- After upgrading to `1.1.3`, new logs will resume using the normal `frpdeck-YYYY-MM-DD.log` naming pattern. Existing abnormal long filenames are not cleaned up automatically in this release; if prior failed rotations left behind oversized filenames or symlink chains, archive or delete those files manually as needed.
