# frpdeck 1.1.2

`1.1.2` is a patch release that closes out the current CLI, sync, HTTP/HTTPS proxy, and MCP cleanup work so the repository is ready for the next release cycle without command-surface drift or release-metadata mismatch.

## Highlights

- Finished the proxy CLI convergence around `proxy import <file>`, `proxy update <name> <file>`, and `proxy add tcp|udp|http|https`.
- Added an explicit top-level `sync` step so rendered output and runtime config are separate, documented phases.
- Completed the HTTP/HTTPS proxy path across config, render, CLI, MCP, docs, and tests.

## Notable Changes

- Server defaults and scaffold output now leave vhost HTTP/HTTPS ports and `subdomain_host` unset unless the user explicitly configures them.
- `sync` is now a first-class top-level command that mirrors managed files from `rendered/` into `runtime/config` without implicitly validating, rendering, reloading, or restarting.
- Proxy maintenance no longer exposes duplicate `proxy validate` / `proxy apply` commands; the proxy CLI is centered on CRUD, import, typed add, and preview.
- MCP proxy tools now align with the CLI mental model: unified `add_proxy`, file import, CRUD, and preview remain; proxy-apply and proxy-validate endpoints are gone.
- Help-text tests were tightened to keep asserting real command content while no longer depending on Typer/Click-specific help exit codes.

## Upgrade Note

If you scripted `proxy update --from-file`, switch those invocations to:

```bash
frpdeck proxy update <name> <file>
```

If you relied on the old flat proxy add commands or proxy-scoped validate/apply commands, update those workflows to the current CLI shape and use top-level `validate`, `render`, `sync`, and `apply` for instance operations.
